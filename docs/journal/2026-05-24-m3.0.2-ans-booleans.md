# M3.0.2 — ANS boolean convention (-1/0, not t/f) (2026-05-24)

The Forth 2012 test suite's `T{ <code> -> <expected> }T` assertions
check literal stack values, so `T{ 5 5 = -> -1 }T` requires `=` to
actually leave `-1` (not Factor's `t`).  This milestone gets us to
that convention without breaking Factor's `if` / `loop` machinery
underneath.

## The two-sided challenge

ANS Forth says:
- Comparators return **-1 for true, 0 for false**.
- IF / WHILE / UNTIL treat **0 as false, anything else as true** —
  matching the "any nonzero value is true" rule.

Factor's native model:
- Comparators return **`t` for true, `f` for false** (Factor's
  singleton booleans, NOT integers).
- `kernel:if` treats only `f` as false-y.  Any other value
  (including the integer `0`!) is truthy.

So a naive "make comparators return -1/0" would break every IF
because Factor's `if` would treat `0` as truthy.

## Solution

Two emit-time transformations:

### 1. Comparators → ANS booleans via wrappers

`factor/forth/runtime/runtime.factor` now defines ans-* wrappers:

```factor
: ans=  ( a b -- n )   =      bool>flag ; inline
: ans<> ( a b -- n )   =  not bool>flag ; inline
: ans<  ( a b -- n )   <      bool>flag ; inline
: ans>  ( a b -- n )   >      bool>flag ; inline
: ans<= ( a b -- n )   <=     bool>flag ; inline
: ans>= ( a b -- n )   >=     bool>flag ; inline
: ans0= ( n -- m )     zero?  bool>flag ; inline
: ans0< ( n -- m )     0 <    bool>flag ; inline
: ans0> ( n -- m )     0 >    bool>flag ; inline
: ansf< ( a b -- n )   <      bool>flag ; inline
: ansf> ( a b -- n )   >      bool>flag ; inline
: ansf= ( a b -- n )   =      bool>flag ; inline
```

`bool>flag` was already defined: `forth-true forth-false ?`.
`?` is Factor's ternary — a conditional move, so the wrappers
fold to a single `cmov` after optimisation.  Zero call overhead.

Resolver entries updated to point at the wrappers:

```rust
("=",   QualifiedBuiltin { vocab: "forth.runtime", factor_name: "ans=" }),
("<>",  QualifiedBuiltin { vocab: "forth.runtime", factor_name: "ans<>" }),
("<",   QualifiedBuiltin { vocab: "forth.runtime", factor_name: "ans<" }),
// ... etc
```

### 2. IF emit converts ANS flag → Factor bool

`emit.rs::Expr::If` now prepends `math:zero?` to the flag and
**swaps the branches** so Factor's `if` runs the right one:

```text
ANS source: flag IF then-body ELSE else-body THEN
Factor IR:  flag math:zero? [ <else> ] [ <then> ] kernel:if
                                                       ^^ if-truthy runs first
                                                       ^^ which is "was 0 = false" = ELSE
```

For `IF ... THEN` without an `ELSE`, emit `unless` (Factor's
inverted `when`):

```text
ANS source: flag IF body THEN
Factor IR:  flag math:zero? [ <body> ] kernel:unless
                                       ^^ runs when input is f
                                       ^^ which is "was non-zero" = ANS true
```

This handles all three cases the ANS spec calls out:
- `-1 IF` (canonical true) → runs THEN
- `0  IF` (canonical false) → skips THEN
- `5  IF` (nonzero, ANS-true) → runs THEN (zero? = f → unless runs)

### What didn't need changing

`BEGIN ... UNTIL` and `BEGIN ... WHILE ... REPEAT` already used
`math:zero?` explicitly in their emit — leftover from the
aspirational comment in runtime.factor that said "the Rust compiler
wraps every comparison".  That comment was right about WHILE/UNTIL;
they happened to be implemented correctly because they emit the
conversion themselves.  IF was the only outlier.

`AND` / `OR` / `INVERT` (bitwise) compose correctly on -1/0 by
construction: `-1 AND -1 = -1`, `-1 AND 0 = 0`, `INVERT -1 = 0`,
`INVERT 0 = -1`.  No changes needed.

## Tests

`tests/session_ans_booleans.rs` — 19 assertions covering all the
T{}T-style cases the Forth 2012 test suite uses:

- Each comparator (`= <> < > <= >= 0= 0< 0>`) tested for both
  true and false outcomes, asserting exact `-1` or `0` strings.
- IF/THEN/ELSE control: confirmed `-1`, `0`, `5`, `-42` all behave
  per ANS spec (only `0` skips THEN).
- IF without ELSE (the `when` path): confirmed both branches.
- AND of two flags, INVERT of each flag — bit semantics preserved.
- Comparator inside a colon def feeding IF — the round-trip from
  `=` through wrapper through emit-time conversion through IF.
