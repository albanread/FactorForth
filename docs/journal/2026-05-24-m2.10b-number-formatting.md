## The user's reframing — again

User's comment that prompted this:

> *"I never liked this Number formatting (<# # #S #>) but its
>  not to bad if you just think of it as more like a small
>  formatting DSL."*

Same pattern as the M2.9 collections discussion.  Look at the
words individually, they're bewildering.  Look at them as a
*tiny DSL with implicit state*, they're a small, clean string-
builder language.

The DSL frames each word as an operation on a buffer:

```
<#       open buffer
   #     append one digit (low-to-high)
   #S    append all remaining digits
   sign  append '-' if negative
   hold  append literal char
#>       close, reverse, yield (c-addr, u)
```

That's it.  ANS made the buffer implicit because Forth has no
"begin/end block" syntax other than these word-pairs.  Once you
see the parallel with modern `with-string-builder [ ... ]`,
the design clicks.

## Implementation

A Factor namespace variable holds the active buffer.  Words build
forward (low-digit first); `#>` reverses at close time.  That's
the inverse of traditional ANS (build backward, return as-is)
but observably identical and easier to reason about.

```factor
SYMBOL: current-num-buf

: <# ( -- ) SBUF" " clone current-num-buf set ;
: #  ( n -- n' ) number-base get /mod digit-chars nth
                 current-num-buf get push ;
: #S ( n -- 0 )  # [ dup 0 = ] [ # ] until ;  ! ≥1 digit always
: sign ( n -- )  0 < [ CHAR: - current-num-buf get push ] when ;
: hold ( ch -- ) current-num-buf get push ;
: #>  ( n -- c-addr u )
    drop current-num-buf get >string reverse
    utf8 encode dup length [ 0 <nf-addr> ] dip ;

: n>$ ( n -- c-addr u ) dup abs <# #S swap sign #> ;
```

PAD doesn't appear anywhere.  No shared temporary.  Each
`<# ... #>` block allocates its own SBUF; GC'd when no
references remain after `#>` returns.

## The lexer paper-cut

First attempt to run `<# # #S #>` produced:

```
malformed number `#S` at line 12, column 16: non-digit character
```

My fault — I'd written the lexer to recognise `#` as an explicit
decimal prefix (`#42` means decimal 42 even if BASE is currently
hex).  But `#`, `#S`, `#>` are user-visible words too.  The fix
was a one-line constraint:

```rust
if let Some(rest) = raw.strip_prefix('#') {
    if !rest.is_empty() && rest.bytes().all(|b| b.is_ascii_digit()) {
        return parse_int_with_base(...);
    }
}
```

`#` prefix only kicks in when the rest is all digits.  `#42` is
still a literal; `#S` falls through to be a word.

This wasn't caught in lex tests because no test happened to
have a token starting with `#` followed by non-digits.  Real
program, real issue.  The journal entry for M2.1 should
probably get a "lessons learned" addendum.

## The output

Demo (`demos/format-numbers.f`):

```forth
1234 n>$ type cr               \ → 1234
-7   n>$ type cr               \ → -7
0    n>$ type cr               \ → 0

." decimal: "
-42 dup abs <# #S swap sign #> type cr

." hex:     "
255 hex
  dup abs <# #S
    120 hold              \ 'x'
    48  hold              \ '0'
  swap sign
  #> type cr
decimal

." binary:  "
binary
13 <# #S #> type cr
decimal
```

Produces:

```
1234
-7
0
decimal: -42
hex:     0xff
binary:  1101
```

Honest ANS formatted output.  The hex prefix block reads
naturally if you know the DSL is "things you call inside push
into the accumulator."  The reverse-at-`#>` is invisible.

## What this clarifies

This is the THIRD ANS construct in a row where the "weird"
form is actually a small DSL once you look at it that way:

- **VARIABLE/CREATE/DOES>** = closure factory + closure body
- **`array` / `farray` / `cbuffer`** = parameterised
  collection templates
- **`<# # #S sign hold #>`** = stack-based string builder

In each case there's a 1979 implementation underneath (PAD,
dictionary memory, data-space pointers) that looks scary, and
a 2026 reframe (closures, type templates, with-string-builder)
that's actually clean.  We translate the 1979 surface into the
2026 implementation and the user gets both.

I notice this is also what the variable-narrowing optimisation
does — translate ANS's "address that escapes into pointer
arithmetic" into "global variable Factor's optimiser can see
through."  Same trick at every layer: read the ANS pattern,
infer what the user *meant*, emit the modern equivalent.
